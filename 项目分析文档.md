# CakeCrash 项目分析文档

## 一、后台服务分析

### 1.1 服务器配置

项目存在服务器配置文件 `assets/resources/dataServers/servers.json`：

```json
[
    { "ip": "niuniu.oriongaming.club", "name": "本机", "port": 43010, "id": 1, "is_test_server": 1, "version": "1.5.6" }
]
```

- 服务器地址：`niuniu.oriongaming.club:43010`
- 标记为测试服务器（`is_test_server: 1`）

### 1.2 网络请求能力

`GameLogic` 中有一个通用的 POST 请求方法（`assets/scripts/frameworks/gameLogic.ts:176-196`）：

```typescript
requestWithPost(url, data, callback)  // 基于 XMLHttpRequest，JSON 格式
```

### 1.3 实际后台使用情况

**当前项目实际上没有接入后台服务。** 具体表现：

| 功能 | 实现方式 | 是否依赖后台 |
|------|---------|-------------|
| 用户登录 | 本地生成随机 userId，游客模式 | 否 |
| 数据存储 | localStorage（Web）/ jsb.fileUtils（原生） | 否 |
| 关卡数据 | 本地 CSV 文件 + JSON 地图配置 | 否 |
| 玩家数据 | `Configuration` 类管理本地持久化 | 否 |
| 时间校验 | 使用 `Date.now()` 客户端本地时间 | 否 |

`playerData` 中虽有 `serverTime` 字段，但实际使用的都是本地时间。`loadSubPackage()` 和 `loadGameSubPackage()` 方法体为空，未做分包加载。

**结论：项目是纯客户端单机游戏，服务器配置和网络请求方法是预留代码，当前未实际使用。**

### 3.2 替换各类素材的方法

#### 3.2.1 替换蛋糕图标（核心玩法素材）

**位置：** `assets/resources/gamePackages/textures/icons/cakes/`

- `cake01.png` ~ `cake08.png` — 8种蛋糕的基础图标
- `cake01Light.png` ~ `cake08Light.png` — 对应的发光/高亮版本

**替换方法：**
1. 准备同名的 PNG 图片，保持相同尺寸
2. 直接覆盖原文件，Cocos Creator 编辑器会自动更新 `.meta` 文件中的引用
3. 该目录有 `AutoAtlas.pac`，编辑器会自动重新打包图集

**代码加载方式：** `resourceUtil.setCakeIcon(cakeName, sprite, callback)`

#### 3.2.2 替换道具图标

**位置：** `assets/resources/gamePackages/textures/icons/props/`

- `001.png` — 锤子 (HAMMER)
- `002.png` — 魔法棒 (MAGIC)
- `003.png` — 刷新 (REFRESH)
- `004.png` — 无限 (INFINITE)

**替换方法：** 同名覆盖即可。

**代码加载方式：** `resourceUtil.setPropIcon(propId, sprite, callback)`

#### 3.2.3 替换背景图

#### 3.2.6 替换角色模型

**位置：** `assets/resources/gamePackages/effects/role/` 和 `assets/loginPackage/effect/woman01/`

- `man01/` — 男性角色（身体、发型、面部部件）
- `woman01/` — 女性角色（身体、发型、头饰、面部部件）

**替换方法：** 同名覆盖对应部件的 PNG 图片。

#### 3.2.7 替换艺术数字

**位置：** `assets/fonts/`

**注意：** README 中记录了艺术数字（labelatlas）在跨设备时可能出现报错的已知问题。替换后可能需要在编辑器中重新生成 labelatlas 资源。

### 3.3 素材替换通用注意事项

1. **保持文件名不变** — 代码中通过文件名引用资源，改名会导致加载失败
2. **保持尺寸一致** — 特别是 UI 元素和图标，尺寸变化会影响布局
3. **不要删除 .meta 文件** — Cocos Creator 通过 .meta 文件维护资源 UUID 引用
4. **AutoAtlas 自动更新** — 带有 `AutoAtlas.pac` 的目录会自动重新打包图集，替换文件后在编辑器中重新构建即可
5. **替换后需在 Cocos Creator 编辑器中打开项目** — 编辑器会自动检测文件变化并更新引用
6. **新增蛋糕种类** — 如果要增加蛋糕种类（超过8种），需要同时修改 `constants.ts` 中的 `LINK_ROWS_COUNT` / `LINK_COLS_COUNT` 以及关卡配置 CSV
